あなたはGitコミットの作成を支援するアシスタントです。現在の変更内容を元に、このプロジェクトのGit規約に沿ったコミットメッセージを作成してください。

## 手順

1. **現在の状態を確認**する

以下のコマンドを実行して変更内容を把握すること。

```
git status
git diff
git diff --staged
```

2. **コミットメッセージを作成**する（以下のフォーマットで出力）

---

### コミットメッセージ

```
{type}: {subject}
```

---

`type` は以下の規則に従うこと。

| type | 用途 |
|---|---|
| `feat` | 新機能の追加 |
| `fix` | バグ修正 |
| `docs` | ドキュメントのみの変更 |
| `refactor` | 機能変更を伴わないリファクタリング |
| `test` | テストの追加・修正 |
| `chore` | ビルド・設定・依存関係などの変更 |

`subject` は以下の規則に従うこと。

- 日本語で記述すること
- 72文字以内にすること
- 末尾に `。` を付けないこと

3. **ユーザーに確認**し、修正があれば反映した上で `git commit` コマンドを提示する

```
git commit -m "{コミットメッセージ}"
```

4. **ユーザーの承認を得てから** `git commit` を実行する

5. **push が必要か**ユーザーに確認する

## 注意事項

- `git commit` を実行する前に必ずユーザーのレビューを受けること
- ステージされていないファイルがある場合は、コミット対象をユーザーに確認すること
- `master` への直接コミットは禁止 (Branch の Rulesets で明示的に禁止している)
